## §1 P0 — actingAgent "强制 JSON 总结" 修复（核心）

### 1.1 修复方案：iteration cap 前一轮注入强制总结指令

**设计哲学**：非侵入式。只在 `iterations === maxIter - 1` 时改变 LLM 的对话上下文，注入一条 User 消息："**你必须立即输出 JSON 格式的执行总结，不要再调用任何工具**"。LLM 应输出纯文本 JSON，被 `iterationText` 捕获并 break。

```typescript
// src/actingAgent.ts 主循环开始处（while 内）
if (!readOnly && iterations === maxIter - 1) {
  // 最后一轮：强制 LLM 输出 JSON 总结
  const forceSummaryMsg = vscode.LanguageModelChatMessage.User(
    `[SYSTEM] You have used all but one of your allowed iterations. ` +
    `You MUST now produce your final structured output. ` +
    `DO NOT call any more tools. ` +
    `Output ONLY the JSON summary of what you have accomplished so far, ` +
    `including partial progress and any actions that were started but not completed. ` +
    `Format: {"executed_actions": [...], "self_assessment": {...}}`
  );
  messages.push(forceSummaryMsg);
  stream.progress(`[${progressPrefix}] acting: forcing JSON summary (iteration ${iterations + 1}/${maxIter})`);
}
```

### 1.2 兜底：撞 cap 时从 capturedToolCalls 构造最小 executed_actions

即使 LLM 仍然调用工具不输出 JSON，runActor 末尾的 fallback 也会构造一个聚合 action：

```typescript
// src/moaCore/runActor.ts 现有 catch 分支已存在
// 增强：当 hitIterationCap 且 output 为空时，把 capturedToolCalls 摘要作为 content
if (result.hitIterationCap && !result.output && result.capturedToolCalls.length > 0) {
  const toolSummary = result.capturedToolCalls
    .map((c, i) => `${i + 1}. ${c.name}(${JSON.stringify(c.input).substring(0, 80)}) → ${c.resultText.substring(0, 200)}`)
    .join('\n');
  executedActions = [{
    action: {
      type: 'inform_user',
      target: 'partial_progress',
      content: `Actor hit iteration cap (${result.iterations}/${maxIterations}). ` +
               `Captured ${result.capturedToolCalls.length} tool calls:\n\n${toolSummary}`,
      rationale: 'LLM did not produce structured JSON; partial progress preserved from capturedToolCalls.',
    },
    status: 'partial',
    output_chars: toolSummary.length,
    error_message: 'Hit iteration cap without producing JSON summary',
    artifacts: [],
  }];
}
```

### 1.3 验收标准

- [ ] 当 LLM 持续调用工具到 cap 时，actor_result.json 至少有 1 条 executed_action
- [ ] 当 LLM 在强制总结轮响应正确时，executed_actions 包含完整的结构化 JSON
- [ ] 不影响正常退出路径（LLM 主动停止调用工具时）

## §3 P1 — III-2 保守执行模式（SafeExecutor 最小版）

### 3.1 v0.19.0 最小实现范围

为避免一次交付过大，v0.19.0 只实现 **manifest 落盘 + 文件备份**，不做用户确认弹窗（推到 v0.20.0）：

```typescript
// src/safeExecutor.ts（新增）
interface SafeActionRecord {
  iter: number;
  type: 'write_file' | 'execute' | 'delete' | 'read' | 'grep' | 'other';
  target: string;
  tool_name: string;
  input_summary: string;  // 截断到 200 字符
  backup_path?: string;   // write_file/delete 时，原文件的 .bak 路径
  status: 'success' | 'failed' | 'partial';
  output_chars?: number;
  timestamp: string;
  error?: string;
}

class SafeExecutor {
  constructor(private taskDir: string, private iter: number) {}
  
  // 拦截所有 tool calls，执行前/后写入 manifest
  async wrapToolCall(
    toolName: string,
    input: unknown,
    invoke: () => Promise<vscode.LanguageModelToolResult>
  ): Promise<vscode.LanguageModelToolResult> {
    const record: SafeActionRecord = {
      iter: this.iter,
      type: this.classifyTool(toolName),
      target: this.extractTarget(input),
      tool_name: toolName,
      input_summary: JSON.stringify(input).substring(0, 200),
      status: 'success',
      timestamp: new Date().toISOString(),
    };
    
    // 预备份：write_file / delete 类，且目标存在
    if ((record.type === 'write_file' || record.type === 'delete') && record.target) {
      try {
        const backup = `${record.target}.bak.${Date.now()}`;
        await fs.copyFile(record.target, backup);
        record.backup_path = backup;
      } catch { /* file may not exist yet */ }
    }
    
    try {
      const result = await invoke();
      record.output_chars = JSON.stringify(result.content).length;
      return result;
    } catch (err) {
      record.status = 'failed';
      record.error = err instanceof Error ? err.message.substring(0, 200) : String(err);
      throw err;
    } finally {
      await this.appendManifest(record);
    }
  }
  
  private async appendManifest(record: SafeActionRecord): Promise<void> {
    const manifestPath = path.join(this.taskDir, 'manifest.json');
    let manifest: SafeActionRecord[] = [];
    try {
      manifest = JSON.parse(await fs.readFile(manifestPath, 'utf8'));
    } catch { /* first record */ }
    manifest.push(record);
    await fs.writeFile(manifestPath, JSON.stringify(manifest, null, 2));
  }
  
  private classifyTool(name: string): SafeActionRecord['type'] {
    if (/write.?file|create.?file|apply.?patch|insert.?edit|replace.?string/i.test(name)) return 'write_file';
    if (/delete|remove|unlink/i.test(name)) return 'delete';
    if (/run.?in.?terminal|exec/i.test(name)) return 'execute';
    if (/read/i.test(name)) return 'read';
    if (/grep|search/i.test(name)) return 'grep';
    return 'other';
  }
  
  private extractTarget(input: unknown): string {
    if (!input || typeof input !== 'object') return '';
    const obj = input as Record<string, unknown>;
    return String(obj.filePath || obj.path || obj.file || obj.target || '');
  }
}
```

### 3.2 接线点

### 3.3 验收标准

```typescript
// src/cacheManager.ts（新增）
class CacheManager {
  // 启动时扫描过期缓存（基于目录 mtime）
  static async cleanupExpired(maxAgeDays: number = 7): Promise<{ removed: string[]; kept: string[] }> {
    const cacheRoot = await getCacheRoot();
    const entries = await fs.readdir(cacheRoot);
    const now = Date.now();
    const cutoff = now - maxAgeDays * 24 * 60 * 60 * 1000;
    
    const removed: string[] = [];
    const kept: string[] = [];
    
    for (const entry of entries) {
      const entryPath = path.join(cacheRoot, entry);
      const stat = await fs.stat(entryPath);
      if (stat.mtimeMs < cutoff) {
        await fs.rm(entryPath, { recursive: true });
        removed.push(entry);
      } else {
        kept.push(entry);
      }
    }
    return { removed, kept };
  }
  
  // 命令面板：MoA: Cleanup Old Tasks (>7 days)
  static async registerCleanupCommand(context: vscode.ExtensionContext): Promise<void> {
    context.subscriptions.push(
      vscode.commands.registerCommand('moa.cleanupOldTasks', async () => {
        const result = await CacheManager.cleanupExpired(7);
        vscode.window.showInformationMessage(
          `MoA: Cleaned ${result.removed.length} task(s) older than 7 days. ` +
          `${result.kept.length} task(s) kept.`
        );
      })
    );
  }
}
```

## §9 教训（补充 final.md §8）

- **L5**：路线图编号体系（用户口述 19/21/22）必须**当场验证物理行号**，不能基于 recon 转述推断；本次实测发现 final.md 的"附录C行号"推断不成立（附录C不存在）
- **L6**：`moa.enableActingAgent`（moaRunner 单次模式）vs `moa.enableActorInLoop`（orchestrator Loop 模式）**语义完全不同**，不能复用同一配置项。复用会造成"我关了开关但 Actor 还在跑"的混乱
- **L7**：actingAgent 的 `finalOutput` 只在 LLM 主动停止工具调用时填充，**iteration cap 退出时 finalOutput 是空字符串**——这是 Layer 2 bug 的根因，必须在 actingAgent 层修复，不能依赖下游 runActor 兜底
- **L8**：MoA 自审对源码细节的描述（如"actorEvidence 未修复"）**可信度有限**，关键修复点必须由主会话直读源码验证
