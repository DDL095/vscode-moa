**版本号变更**:

## §2 P0 — 核心改造(必做,版本发布的硬门槛)

### P0-1 Planner mini-loop + role_setup

**现状**:[src/moaCore/runPlanner.ts](../../src/moaCore/runPlanner.ts) 单次调用,无迭代,无 role_setup 输出

**目标**:

- mini-loop 默认最多 5 次,绝对上限 20 次(配置:`moa.plannerMaxIterations`)
- plan_coverage 阈值 0.9(配置:`moa.plannerCoverageThreshold`)
- iter 2+ 可调 read-only 工具(read_file / list_dir / grep_search / get_errors)
- 每轮最多 3 次工具调用(防过度探索)
- plan_coverage < 0.5 且 iter ≥ 2 → 触发 ask_user
- 输出新增字段:`role_setup` / `plan_coverage` / `needs_replan` / `ask_user` / `ask_user_questions` / `process_language` / `task_type`
- 完整提示词见 [../planner-system-prompt.md](../planner-system-prompt.md)

**实施**:

- 重写 `src/moaCore/roles.ts` 的 `buildPlannerPrompt()`(签名扩展,接收 systemContext)
- 重写 `src/moaCore/runPlanner.ts`(实现 mini-loop + ask_user)
- 扩展 `PlannerOutput` schema(7 个新字段,带默认值保证向后兼容)
- 新增 `src/moaCore/plannerMiniLoop.ts`(或合并到 runPlanner.ts)

**配置项**(package.json):

```json
{
  "moa.enablePlannerIteration": {
    "type": "boolean",
    "default": true,
    "description": "Enable Planner mini-loop with plan_coverage convergence. Disable for v0.21.x single-call behavior."
  },
  "moa.plannerMaxIterations": {
    "type": "number",
    "default": 5,
    "minimum": 1,
    "maximum": 20,
    "description": "Max Planner mini-loop iterations. 1 = single-call mode (v0.21.x compatible)."
  },
  "moa.plannerCoverageThreshold": {
    "type": "number",
    "default": 0.9,
    "minimum": 0.5,
    "maximum": 1.0,
    "description": "plan_coverage threshold for Planner convergence."
  },
  "moa.plannerAllowTools": {
    "type": "boolean",
    "default": true,
    "description": "Allow Planner to call read-only tools (read_file/list_dir/grep_search/get_errors) in iter 2+."
  }
}
```

**工作量**:2-3 天

### P0-2 systemContext.ts(基础设施层注入)

**现状**:无统一入口;`workspaceContext.ts` 存在但有 dead code bug([src/moaRunner.ts](../../src/moaRunner.ts) L320-325)

**目标**:新建 `src/systemContext.ts`,提供 4 段动态注入:

| 注入项                 | 内容来源                                                                                  |
| ---------------------- | ----------------------------------------------------------------------------------------- |
| `ENV_CONTEXT`        | `workspaceContext.ts` 扩展(activeFile / openDocs / workspaceFolders / projectTree / gitRoot / instructionFiles 清单) |
| `TOOL_EFFICIENCY`     | 静态模板(工具调用纪律)                                                                  |
| `CUSTOM_INSTRUCTIONS` | `instructionScanner.ts` 扫描 7 路径(**不截断**)                                          |
| `RUNTIME_INSTRUCTIONS`| `instructionScanner.ts` 扫描 4 文件夹 SKILL.md frontmatter                               |

**实施**:

```typescript
// src/systemContext.ts
export interface SystemContext {
  envContext: string;
  toolEfficiency: string;
  customInstructions: string;
  runtimeInstructions: string;
}

export async function buildSystemContext(): Promise<SystemContext>;
export function renderForRole(ctx: SystemContext, role: 'planner' | 'recon' | 'actor'): string;
export function renderEnvBrief(ctx: SystemContext): string;  // 给 Refs/Aggregator 的精简版(可选)
```

**工作量**:1-2 天

### P0-3 instructionScanner.ts(指令与 skill 扫描)

**现状**:不存在

**目标**:扫描 7 路径的指令文件 + 4 文件夹的 skills,**不截断**(用户明示)

**7 路径**(指令文件):

```
workspace:
  ./CLAUDE.md
  ./CLAUDE.local.md
  ./AGENTS.md
  ./.claude/CLAUDE.md
  ./.github/copilot-instructions.md

user home:
  ~/.claude/CLAUDE.md
  ~/.copilot/copilot-instructions.md
```

**4 文件夹**(skills):

```
workspace:
  ./.github/skills
  ./.claude/skills

user home:
  ~/.copilot/skills
  ~/.claude/skills
```

**缓存策略**:按文件 mtime 缓存,mtime 未变则用缓存

**实施**:

```typescript
// src/instructionScanner.ts
export interface InstructionFile {
  path: string;
  relativePath: string;
  sizeKb: number;
  source: 'workspace' | 'home';
  content: string;  // 完整内容,不截断
}

export interface SkillEntry {
  name: string;          // 从 SKILL.md frontmatter
  description: string;   // 从 SKILL.md frontmatter
  folder: string;
  hasScripts: boolean;
}

export interface ScanResult {
  instructions: InstructionFile[];
  skills: SkillEntry[];
  scanDurationMs: number;
}

export async function scanInstructionsAndSkills(): Promise<ScanResult>;
```

**安全考虑**:

- 单文件大小超过 200KB 时给 Planner 警告(但仍传入,由 Planner 决定如何处理)
- 检测疑似敏感信息(API key / token 格式)给 Planner 警告(不删减)
- 不递归读取 skill 内的脚本文件(只读 SKILL.md)

**工作量**:1-2 天

### P0-4 Recon Aggregator 始终运行 + 双轨制

**现状**:[src/moaCore/runRecon.ts](../../src/moaCore/runRecon.ts) 的 `callRecon` 已实现 Recon Aggregator,但单 Recon 时跳过(mode='single' 时无整合)

**目标**:

- **始终运行** Recon Aggregator(单 Recon 时也跑,做标准化)
- 原始 Recon 产出**始终落盘**(便于复盘):`.moa_cache/<task_id>/iteration_N/recon/recon_<label>.json`
- 整合后产出落盘:`.moa_cache/<task_id>/iteration_N/recon_result.json`
- **双轨制角色设定**:
  - 默认模式:内置静态 prompt(`buildReconAggregatorPrompt`,硬编码)
  - Planner 驱动模式:Planner 的 role_setup.recon_aggregator 覆盖默认

**实施**:

- 修改 `callRecon` 逻辑:无论 mode 是 single 还是 parallel,都跑 Recon Aggregator
- 单 Recon 时,Recon Aggregator 做"格式标准化"(去重 / 排序 / 引用统一),不强行合并
- 新增 `buildReconAggregatorPrompt`(静态默认)+ 支持 role_setup 注入(Planner 驱动)
- 新增配置:`moa.reconAggregatorMode`(default / planner)

**配置项**:

    "type": "number",

## §7 风险与缓解

| 风险                                              | 严重度 | 缓解措施                                                                                                |
| ------------------------------------------------- | ------ | ------------------------------------------------------------------------------------------------------- |
| Planner mini-loop 不收敛(20 次都不到 0.9)       | 高     | 硬上限 20 次 + ask_user 触发(plan_coverage < 0.5)                                                       |
| Planner 成本过高(5 次 mini-loop)                | 中     | 默认 5 次,简单任务通常 1-2 次收敛;用户可降 `moa.plannerMaxIterations`                                  |
| role_setup 格式错误(缺字段 / 类型错)            | 中     | JSON schema 校验 + 默认值(缺则用 v0.21.x 静态 prompt fallback)                                       |
| instructionScanner 扫描慢(大 workspace)          | 中     | mtime 缓存 + 异步扫描(不阻塞主流水线)                                                                |
| CLAUDE.md 含敏感信息(API key)                   | 中     | 检测 + 警告(不删减,让 Planner 决定)                                                                   |
| Recon Aggregator 始终运行增加成本                | 低     | 单 Recon 时用最小 prompt;Aggregator 模型可配置为便宜模型                                               |
| 向后兼容性(老用户升级)                          | 高     | `moa.enablePlannerIteration` 开关(默认 true,可关闭回退到 v0.21.x 单次模式)                            |
| 多 root workspace 的 ENV_CONTEXT 取哪个          | 低     | 默认取第一个 root,所有 root 的 instructionFiles 都扫描                                                |
| Planner process_language 误判                    | 中     | 启发式 fallback(从 user prompt 字符统计推断)                                                          |
| role_setup 让 Refs 失去多模型可比性              | 高     | **架构红线**:Refs 的 role_setup 只有 hint 一句话,不改 JSON schema(见设计哲学 §3.3)                  |
