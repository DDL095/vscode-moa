## 0. 一句话目标

**修复 MoA 审查（v0.17 loop, iter=2, confidence=88%）发现的 5 类文字与提示不一致问题，使 package.json / README.md / 代码注释 / NLS 文件 / 帮助提示 与实际 8 步流程 + v0.18 配置项默认值 完全对齐。**

设计原则：**只改文字，不改行为**。所有修复点都是用户可见的描述/注释/默认值表格，不涉及任何运行时逻辑。编译产物（`dist/extension.js`）会因字符串变化而重新打包，但语义零变化。

## 1. 背景与问题清单

MoA 审查（final.md）发现 5 类问题，按严重度排序：

### P0 — Configure Models 步数多重矛盾（用户直接看到）

| 位置 | 当前值 | 应为 | 来源 |
|---|---|---|---|
| `package.json` L126 `moa.configureModels.description` | `v0.14.14: Five-step flow — (0/4)...(4/4)` | `v0.18.2: Eight-step flow — (0/7)...(7/7)` | 显示在 Command Palette |
| `README.md` L116 | `an 8-step flow` | ✅ 正确 | — |
| `README.md` L294 | `the 5-step guided flow` | `the 8-step guided flow` | 与 L116 矛盾 |
| `README.md` L341 文件布局注释 | `Configure Models 5-step flow` | `Configure Models 8-step flow` | 与 L116 矛盾 |
| `src/moaConfig.ts` L292 顶部注释 | `v0.14.14 Step 0/4` | `v0.18.2 Step 0/7` | 内部注释 |
| `src/moaConfig.ts` L379 内联注释 | `Step 2/4: aggregator` | `Step 2/7: aggregator` | 内部注释（`title:` 字符串 L399 已是 2/7 正确） |
| `dist/extension.js` (编译产物) | `Open Step 0-4 flow` | 重新编译后自动变 | 不手改 |

**根因**：v0.18.2 把流程从 5 步扩到 8 步，改了 `title:` 字符串（用户在 QuickPick header 看到的）但漏改了 `package.json` 的 command description 和 README 中的散文描述。

### P1 — 配置项默认值偏差（用户按文档配置会得到错误预期）

| 配置项 | README 标注 | 实际值（package.json + 代码） | 偏差 |
|---|---|---|---|
| `moa.reconL3Threshold` | `30000` | `200000` | **6.7×** |
| `moa.reconContextChars` | `30000` (描述为 "Character budget") | `500000` (DEPRECATED，仅审计) | **16.7×** + 语义冲突 |
| L3 输出目标字符数（散文） | `~5k chars`（L228 + L210 表格） | `DEFAULT_TARGET_CHARS = 50000` | **10×** |

**根因**：v0.14.4/v0.14.5 调大了这些默认值（"1M 上下文模型可以消化"），但 README 表格未同步。**注意**：`src/cacheReadme.ts` L195 的 cache README 模板里写的是 `200000`（正确），所以用户在 `.moa_cache/README.md` 看到的是对的，但在主 README 看到的是错的——构成 README 内部矛盾。

### P2 — 路线图状态滞后（`docs/roadmap/v0.15.0-closed-loop-moa.md`）

- 头部仍标 `状态：设计草案（design draft），尚未开始实施`
- §1.1/§1.2 仍把"Path A 无反馈路径、Path B 无 acting agent"列为"v0.15.0 待解决的关键缺陷"
- 但 §15（追加章节）已记录 v0.15.0 完整实施 + 两个 hotfix
- §10 文档更新清单仍全部 `[ ]` 未勾选

**根因**：v0.15.0 发布时只追加了 §15 实施记录，没回头改前面章节的"计划"语气为"已完成"语气。

### P3 — 帮助提示版本号滞后（`src/moaHandler.ts` L190）

```typescript
function buildHelpMarkdown(): string {
  return [
    "## MoA Bridge — Usage (v0.17.0)",  // ← 当前 package.json 已是 0.18.2
```

**根因**：硬编码版本号字符串，每次发版都要手改。短期：同步到 0.18.2；长期：从 package.json 读。

### P4 — NLS 本地化文件严重过时（`package.nls.json` / `package.nls.zh-cn.json`）

- 两个字段集完全不对应实际命令/配置项
- 例如 `command.moa.preset.title`、`configuration.defaultPreset`、`configuration.presetsDir`、`configuration.timeoutSec` 都是早期草稿遗留，与 v0.18 实际命令（`moa.configureModels` / `moa.switchPreset` / `moa.probeTools`）和配置项（`moa.presets` / `moa.refModels` / ...）完全不匹配
- **影响有限**：因为 `package.json` 没用 `%key%` 语法引用这些 NLS key，所以实际不显示给用户
- 但留着是"未来某人按 NLS 添加 % 引用"的陷阱

**根因**：v0.14~v0.18 大改命令/配置 schema 时，NLS 文件从未同步。

## 5. 风险评估

| 风险 | 概率 | 影响 | 缓解 |
|---|---|---|---|
| 字符串替换误伤（多匹配） | 低 | 低（git diff 可视） | 每个替换都用 3-5 行 context `replace_string_in_file` |
| README 表格列对齐破坏 | 中 | 极低（Markdown 渲染容错） | 替换后用 grep 检查表格行数 |
| 编译失败 | 极低 | 低 | 字符串字面量改动不可能触发 TS 错误 |
| 用户配置被覆盖 | 0 | 0 | 不动 `configuration` 默认值，只动 README 描述 |
